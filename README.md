## 🎯 What is Gain Standard Beckhoff Framework (GSBF)?

This repository provides unified access to the GSBF source code, compiled libraries (TwinCAT 4026), and complete documentation, including example projects and how-to guides.

**📦 Download Compiled Libraries:** Access the latest compiled GSBF libraries from the section above.

**🔓 Source Code Access:** Interested in the source repositories (GSBF_Common, GSBF_HMI, GSBF_License, GSBF_Camming, GSBF_FlyingSaw)? [Contact us](mailto:GSBF@gain.nl) to discuss Accessibility options.

The **GSBF** is a comprehensive set of reusable PLC components, HMI controls, and automation tooling built on ISA-88 equipment models and PackML state machines. It provides:

- **PLC architecture** – Hierarchical equipment models and standardized control layer design
- **Naming & coding conventions** – Consistent patterns for maintainability across teams
- **Reusable libraries** – Pre-built software components ready for your TwinCAT 4 projects
- **Full documentation** – Every component, class, and interface fully documented
- **Example projects** – Real-world demonstrations of GSBF in action
- **How-to guides** – Step-by-step tutorials for common tasks

It is designed to shorten engineering cycles, reduce integration risk, and keep projects consistent and maintainable.

### Why Use GSBF?

- **Faster delivery** – PackML-ready units and equipment modules shorten engineering time.
- **Lower risk** – Proven control modules and utilities reduce integration surprises.
- **Consistent quality** – Standardized ISA-88 layering keeps solutions maintainable and scalable.
- **Reuse everywhere** – Shared services (math, motion, file handling) keep solutions lean.
- **Built-in governance** – Auxiliary utilities for communication, diagnostics, and metadata add traceability.

For a practical example of how GSBF is used in real projects, see the [ExampleProject](https://github.com/Gain-Automation-Technology/ExampleProject).
